# Expedition Hub PDF Slide Copy

## Core Message

Expedition Hub는 원정대 구성부터 제출 준비 관리, 결과 확인까지 이어지는 재사용형 해커톤 실행 포털입니다.

## Slide Notes

### Slide 2
- 단순 소개 페이지가 아니라 실제 준비 흐름을 따라갈 수 있게 만들었습니다.
- 참가자가 원정대를 구성하고 제출 준비를 관리하는 실행 단계까지 제품 안에 담았습니다.
- 심사자는 배포 URL만으로 핵심 구조를 확인할 수 있습니다.

### Slide 3
- 공개 포털은 탐색과 이해에 집중합니다.
- Camp는 원정대 구성을, 팀 전용 War Room은 제출 준비 관리 허브 역할을 맡습니다.

### Slide 6
- 상세 페이지는 개요, 안내, 평가, 일정, 상금, 팀, 제출, 리더보드의 8개 섹션을 유지합니다.
- 사용자 흐름이 끊기지 않도록 `원정대 만들기`, `팀 찾기`, `제출 준비`, `결과 확인`을 연결했습니다.

### Slide 8
- War Room은 베이스캠프 요약과 원정대 실행 워크플로우를 한 화면에 모았습니다.
- 단계 전환, 카드 이동, 체크리스트, 링크, 메모를 로컬 상태로 유지합니다.

### Slide 9
- 공개 상세에서는 최소 draft만 저장하고, 실제 원정대 단위 제출 준비 관리는 War Room으로 이어집니다.
- 심사자 입장에선 공개 페이지와 팀 실행 구조를 동시에 설명할 수 있습니다.

### Slide 11
- 검수 기준에 맞춰 외부 키 없이 접속 가능한 구조를 유지했습니다.
- 배포 URL과 GitHub 링크로 실행/설명/검증이 모두 가능하도록 정리했습니다.

### Slide 12
- 폰트는 라이선스 근거를 명시하고, 이미지와 SVG 아이콘은 팀 직접 생성 자산으로 정리합니다.
- 제출 자료는 실제 배포 화면 캡처를 우선 사용합니다.

### Slide 13
- Expedition Hub는 원정대 구성, 제출 준비 관리, 결과 확인을 하나의 실행 흐름으로 연결합니다.
- 자동화된 코드 리뷰와 문서 정합성 점검 흐름을 운영해 회귀와 불일치 가능성을 줄였습니다.
- 심사자는 배포 URL과 GitHub 링크만으로 구조와 품질 관리 방식을 함께 확인할 수 있습니다.
