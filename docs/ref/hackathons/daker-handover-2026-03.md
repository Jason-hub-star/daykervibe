## Rules
### Product / Tech
- 웹페이지 형태의 서비스를 개발해야 한다.
- 제공된 자료를 기반으로 웹페이지를 완성해야 한다.
- 자료를 바탕으로 팀의 아이디어로 확장 기능과 UX 개선을 제안할 수 있다.
- 개발 도구는 제한 없이 바이브 코딩 툴을 활용할 수 있다.
- 배포는 Vercel이 필수다.

### Deployment / Data
- 배포 URL은 외부에서 접속 가능해야 하며 심사 기간 동안 접근 가능해야 한다.
- 예시 자료 외 데이터는 제공되지 않는다.
- 더미 데이터와 `localStorage`를 활용해 구현할 수 있다.
- 필요한 정보는 팀이 더미 데이터로 구성하고, 변경 사항은 로컬에 저장되어 유지되도록 구현하면 된다.
- 외부 API 또는 외부 DB를 쓰더라도 심사자가 별도 키 없이 확인 가능해야 한다.
- 키가 필요한 기능은 평가에서 확인이 제한될 수 있다.

### Copyright / Fairness
- 제출물의 코드, 이미지, 폰트, 아이콘 등은 저작권과 라이선스를 준수해야 한다.
- 타인의 결과물을 무단 복제, 도용하거나 라이선스를 위반하면 실격 또는 감점될 수 있다.
- 팀 내 역할 분업은 자유다.
- 생성형 AI 도구 사용은 허용된다.

### Judging Notice
- 심사는 기본적으로 배포 URL을 기준으로 진행한다.
- 확인이 어려운 기능은 심사에서 제외될 수 있다.
- 운영진 추가 안내가 있으면 공지 내용이 우선한다.

## Product Spec
### Common Rules
- 모든 페이지 상단바에서 `메인`, `/hackathons`, `/camp`, `/rankings` 이동이 확실히 되어야 한다.
- 공통 상태 UI 3종은 모든 주요 페이지에서 보여준다.
  - `로딩중...`
  - `데이터 없음`
  - `에러`
- 기본 데이터 저장 전략은 `localStorage` 우선이다.

### `/`
- 큰 카드 또는 큰 버튼 3개를 배치한다.
- 항목:
  - 해커톤 보러가기 -> `/hackathons`
  - 팀 찾기 -> `/camp`
  - 랭킹 보기 -> `/rankings`

### `/hackathons`
- 해커톤 목록을 리스트 또는 카드형으로 길게 노출한다.
- 각 항목 기본 정보:
  - 제목
  - 상태 (`진행중`, `종료`)
  - 태그
  - 시작일
  - 종료일
  - 참가자수
- 카드 클릭 시 `/hackathons/:slug`로 이동한다.
- 옵션:
  - 상태 필터
  - 태그 필터
- 참가자수는 데이터가 없으면 비노출 또는 대체 표기 처리한다.

### `/hackathons/:slug`
- 핵심 페이지다.
- 아래 섹션은 필수다.
  - 개요 (`overview`)
  - 안내
  - 평가 (`eval`)
  - 일정 (`schedule`)
  - 상금 (`prize`)
  - 팀 (`teams`)
  - 제출 (`submit`)
  - 리더보드 (`leaderboard`)

### Teams Section
- 해커톤 상세 페이지 안에서 `이 해커톤 팀 구성` 흐름을 제공한다.
- 기본 팀 목록 정보:
  - 팀명
  - 소개
  - 모집중 여부 (`isOpen`)
  - 모집 포지션 (`lookingFor[]`)
  - 연락 링크 (`contact.url`)
- 옵션 기능:
  - 초대
  - 수락
  - 거절
- 팀 구성 시 유의사항 팝업이 있으면 좋다.

### Submit Section
- 제출 가이드를 화면에 보여준다.
- 최소 제출 폼을 둔다.
- 필드:
  - 메모 (`notes`, optional)
  - 파일
- 허용 형식은 해커톤 설정에 따라 다르게 처리한다.
  - 예: `zip`, `pdf`, `csv`
- 현재 이 대회의 공식 제출 구조는 `text/url/pdf`다.

### Leaderboard Section
- 점수 기준을 설명해야 한다.
- 해커톤 설정에 따라 계산 방식이 달라질 수 있다.
  - 투표 기반
  - 평가 산식 기반
- 참가했지만 제출 내역이 없으면 순위 대신 `미제출`로 표기한다.

### `/rankings`
- 해커톤 전체 참여와 순위 기록을 합산한 유저별 글로벌 랭킹 테이블을 제공한다.
- 최소 컬럼:
  - `rank`
  - `닉네임`
  - `points`
- 옵션:
  - 기간 필터 `최근 7일 / 30일 / 전체`

### `/camp`
- 팀원 모집 게시판 성격의 화면이다.
- 해커톤과 연결되지 않아도 생성 가능해야 한다.
- 해커톤 상세에서도 팀 생성이 가능해야 한다.
- 쿼리 필터:
  - `/camp?hackathon=:slug` -> 해당 해커톤 팀만 보기
- 옵션:
  - 채팅 또는 쪽지 보내기 기능

### Camp Form
- 팀명: 필수
- 소개: 필수
- 모집중 여부 (`isOpen`)
- 모집 포지션 (`lookingFor[]`)
- 연락 링크 (`contact.url`)
  - 오픈카톡
  - 폼
- 지원 기능:

- `hackathons`
- `teams`
- `submissions`
- `leaderboards`

### Required vs Optional
- 필수 구현:
  - 상단바
  - 상태 UI 3종
  - 메인
  - 해커톤 목록
  - 해커톤 상세 필수 섹션
  - 캠프 기본 모집
  - 제출 최소 흐름

  - 글로벌 랭킹 기본형

  - 상태/태그 필터

- 수상 시 이름과 초상이 홍보 목적으로 사용될 수 있다.
- 개인정보는 운영 목적에 따라 수집, 저장, 공유될 수 있다.
- 제출물은 고유 저작물이어야 하며 제3자 권리를 침해하면 안 된다.
- 시스템 오류, 인터넷 문제, 운영 변경 등으로 대회가 취소, 수정, 중지될 수 있다.

### 11. 법적 준거
- 한국 법이 적용된다.
- 분쟁은 서울 관할 법원에서 처리된다.
- 대회 참여나 수상은 고용 제안이 아니다.
