## Summary
Notebook 본문은 아직 직접 반영되지 않았다. 현재 이 문서는 로컬 JSON, `memo.png`, `Hackathon-UI-Flow.png`, 그리고 사용자가 직접 적은 페이지별 웹 기능 명세를 기준으로 업데이트된 상태다.

## Must Build
- 공통 상단바 라우트: `/`, `/hackathons`, `/camp`, `/rankings`
- 공통 상태 UI: `로딩중...`, `데이터 없음`, `에러`
- 메인 페이지 3개 진입 카드
- 해커톤 목록 페이지
- 해커톤 상세 핵심 섹션
  - 개요
  - 안내
  - 평가
  - 일정
  - 상금
  - 팀
  - 제출
  - 리더보드
- 팀 모집 페이지
- 글로벌 랭킹 페이지
- 기본 저장 전략: `localStorage`

## Must Not Assume
- 서버나 외부 DB가 반드시 필요하다고 가정하지 않는다
- 추가 실데이터 제공이 있을 것이라고 가정하지 않는다
- 심사자가 API 키를 입력해 줄 것이라고 가정하지 않는다
- 다른 팀의 내부 정보나 비공개 유저 정보를 노출해도 된다고 가정하지 않는다

## Submission Checklist
- 기획서 텍스트 또는 URL 준비
- 외부 공개 웹 URL 준비
- PDF URL 준비
- 제출 가이드 노출
- 제출 폼 최소 필드 반영
- 참가했지만 제출 내역 없는 경우 `미제출` 표기 처리
- 심사자가 로그인 없이 확인 가능한지 검수

## Demo Story
- 메인에서 원하는 진입점을 바로 고른다
- 해커톤 목록에서 특정 해커톤을 찾고 상세로 진입한다
- 상세 페이지에서 개요, 일정, 평가, 팀 구성, 제출, 리더보드 흐름을 한 번에 확인한다
- 캠프에서 팀 모집글을 보거나 작성한다
- 랭킹에서 전체 참여 포인트를 확인한다

## Manual Spec Items
- 메인 페이지 카드 3종
- `/hackathons` 목록에서 상태/태그 필터 옵션
- `/hackathons/:slug`가 실질적 중심 페이지
- `teams`는 해커톤과 무관하게도 생성 가능
- `submit`는 notes optional과 해커톤별 파일 유형 처리 필요
- `leaderboard`는 계산 방식 설명과 `미제출` 표기 필요
- `/rankings`는 글로벌 유저 랭킹 기준

- 유저가 비공개한 정보 공개 금지
- 다른 팀의 내부 정보 공개 금지
